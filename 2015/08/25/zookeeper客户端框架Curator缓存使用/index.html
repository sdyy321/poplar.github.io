一般我们在查询zookeeper数据时如果每次都得请求服务器会严重影响我们的系统性能，为了解决集群中每个节点的数据查询，zookeeper提供了缓存功能，本文为你介绍zookeeper提供的三种缓存方式

三种缓存方式

zookeeper使用的缓存类似监听器的功能，你可以设置监听器来监听服务器的数据并同步到本地实现缓存。

  • Node Cache
    监听一个节点自身的状态变更
  • Path Cache
    监控一个节点的所有子节点的状态变更
  • Tree Cache
    监控一个节点自身以及所有子节点的状态变更，你可以看成是上面Node Cache和Path Cache的组合

你可以根据自己的需要来选择合适缓存类型，下面我们基于curator客户端为大家一一介绍
首先添加项目依赖，Cache功能属于Curator-Recipes项目提供的，所以我们除了添加curator-framework依赖外还需要添加curator-recipes依赖才能使用Cache。

Path Cache

上面测试用例中，我们监控了/test节点，执行pathCacheTest测试输出如下：

从结果中我们可以发现

1、只有添加、删除子节点和修改子节点数据才会触发监听器，对父节点的操作不会触发监听器，对子节点的权限修改也不会触发。

2、cache.start(PathChildrenCache.StartMode.POST_INITIALIZED_EVENT);会在本地cache初始化后会同步远程数据并发送一个INITIALIZED事件告知监听器，除了POST_INITIALIZED_EVENT模式启动还有NORMAL，该模式启动本地cache初始化后不会同步远程数据，BUILD_INITIAL_CACHE这个模式启动本地cache初始化后会同步远程数据，但不会发送INITIALIZED事件通知监听器

Tree Cache

2、在cache初始化后会发送一个INITIALIZED事件，表示本地缓存已经初始化完成
